# SafeTrace v0.9 — Security and Governance Readiness

This module turns SafeTrace's ethical boundaries into executable release gates.

It provides:

- explicit role-based permissions with default-deny behaviour;
- a hash-chained audit log that detects altered historical events;
- machine-readable controls with owners, evidence and blockers;
- separate readiness decisions for synthetic evaluation and restricted partner use;
- a hard ban on autonomous publication, referrals and guilt decisions.

## Current decision

- **Synthetic/public evaluation:** ready.
- **Restricted live partner pilot:** blocked.

The live pilot remains blocked until there is a named qualified partner, production security architecture, independent security review, partner-specific privacy documentation and qualified legal/editorial approval.

This is a strength, not a missing disclaimer: the system enforces its own boundary instead of quietly treating a portfolio prototype as a production evidence service.
